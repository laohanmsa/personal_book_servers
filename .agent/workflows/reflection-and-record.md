# Reflection And Record

Use this workflow near the end of a meaningful session.

## Steps

1. Inspect the session's user requests and completed work.
2. If git is available, inspect file changes and commit sequence; otherwise use the visible file edits and command evidence.
3. Identify the user's direction-level intent and why their choices matter.
4. Identify agent patterns worth preserving: debugging method, implementation discovery, validation lessons, workflow improvements.
5. Suggest concise experience entries, rule improvements, or workflow updates.

## Rules

1. Do not add rules or workflows from this workflow unless the user explicitly asks.
2. Do not record sensitive user data, secrets, or private identifiers.
3. Separate verified lessons from guesses.
